Interfaz & Documentación

El Reto

  • 1,000,000+ de títulos (Goodreads) cargados localmente.
  • Respuesta < 50 ms para una sensación "instantánea".
  • Evitar duplicados y ruido (múltiples ediciones de un mismo libro).
  • Permitir dos modos de orden: "Más buscados" (popularidad) y A-Z al instante.
  • Sin servidor: todo corre en un Web Worker ⇢ cero latencia de red.

Idea de Alto Nivel

1. Carga: Usuario carga CSV ⇢ Barra de progreso en UI.

2. Indexación: Worker normaliza, deduplica y crea un Índice Trie.

3. Búsqueda: Usuario escribe ⇢ se envían tokens completos + prefijo actual.

4. Candidatos: El Trie devuelve candidatos en O(L).

5. Ranking: Intersección rápida de resultados + Heap para obtener el top-10.

6. Renderizado: Sugerencias se pintan en la UI en tiempo real.

7. Feedback: Clic en sugerencia ⇢ contador de popularidad ++.

Flujo Técnico

  1. Carga de CSV por streams en chunks de 20 KB.
  2. normalize() quita tildes, signos y pasa a minúsculas.
  3. UnionFind.union() agrupa ediciones bajo un clusterId único.
  4. trieInsert(token, clusterId) con contador hits = 0.
  5. Búsqueda con lookupWordExact y lookupPrefixCompletions.
  6. Intersección de conjuntos (Set ∩ Set) → universo de candidatos.
  7. Heap.push(-hits, clusterId) preserva el top-10 de popularidad.
  8. UI rescata metadatos del cluster en O(1) y pinta la lista.

Rendimiento

lookupWordExact: 0.01 ms promedio.

lookupPrefixCompletions: 15 ms peor caso (1M títulos).

Valor para el Usuario

  • Experiencia "Google-like" sin servidores caros.
  • Escalabilidad eficiente: tiempo de búsqueda no crece linealmente.
  • Modo 100% offline, ideal para PWAs y entornos sin conexión.
  • Arquitectura modular para futuras mejoras (stemming, sinónimos).
